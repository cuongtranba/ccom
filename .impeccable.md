## Design Context

### Users
Primary audience: Duke (the boss) and team leads evaluating the Inventory Network Protocol. They need to immediately grasp that this is a working system, not a theoretical framework. They're technical decision-makers who value proof over polish.

### Brand Personality
**Autonomous. Neural. Alive.**
The interface should feel like looking into a living network — nodes firing, signals propagating, decisions emerging from consensus. It should convey that this isn't documentation — it's infrastructure that thinks.

### Aesthetic Direction
- **Visual tone**: Sci-fi neural network — animated nodes, signal pulses traveling along traces, state transitions visualized in real-time
- **Theme**: Dark mode. The network lives in darkness; signals and state changes are the light.
- **Color approach**: Deep dark base with warm signal colors (amber/gold for proven, red for suspect/broke, cool blue for unverified, green pulses for propagation). NOT neon. NOT purple gradients. Think bioluminescence, not cyberpunk.
- **Typography**: Technical but not monospace-default. Sharp, engineered display font for headings. Clean proportional for body.
- **References**: Neural network visualizations, CERN data visualizations, dark observatory interfaces
- **Anti-references**: Generic SaaS landing pages, AI slop (purple gradients, glassmorphism), corporate blue-gray, minimalist safe design

### Design Principles
1. **Show, don't describe** — Animate the actual state machine, don't just list features
2. **The network is the hero** — The interactive node visualization IS the landing page, not decoration beside text
3. **Signal over noise** — Every visual element should represent real data flow, not decoration
4. **Dangerous means alive** — The UI should feel like it could change at any moment because it's connected to real state
5. **Proof over assertion** — Show the CLI output, the actual commands, the real state transitions
