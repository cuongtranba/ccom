## Commands

```bash
# Tests
bun test                                    # full suite
bun test packages/node                      # node tests only

# Dev
bun run server          # start central WebSocket server
```

## Architecture

**Monorepo** with 4 packages: `shared` (types + protocol), `server` (WebSocket hub), `node` (local client + MCP), `dashboard` (Astro SSR UI).

### Message Flow

```
Claude ‚Üí MCP stdio ‚Üí channel.ts ‚Üí WSClient ‚Üí Central Server (Redis-backed) ‚Üí Other Nodes
```

- `channel.ts` ‚Äî MCP server exposing 3 tools via `@modelcontextprotocol/sdk` stdio transport
- `ws-client.ts` ‚Äî WebSocket with exponential backoff reconnect
- `ws-handlers.ts` ‚Äî forwards inbound envelopes from server to EventBus
- `log-buffer.ts` ‚Äî in-memory ring buffer (200 entries) with `setOnPush` callback hook for SSE streaming

### Server Routing

Central server (`packages/server`) routes envelopes:
1. Target node online on this instance ‚Üí direct `ws.send()`
2. Online on another instance ‚Üí Redis pub/sub
3. Offline ‚Üí Redis outbox queue, drained on reconnect

### Admin Real-Time Stream

The server exposes `GET /api/stream` (SSE, `?key=<ADMIN_KEY>`) that pushes two event types:
- `signal` ‚Äî every routed envelope: `{ from, to, project, type, content, timestamp }`
- `log` ‚Äî every `LogBuffer` entry as it's pushed

`RedisHub.onRoute` and `LogBuffer.setOnPush` are the hooks that feed the broadcast. The admin UI (`packages/admin`) connects via `useSignalStream` hook on `EventSource`, feeds signals into the `SignalFlow` React Flow graph (real-time node topology with animated edges), and populates the live log stream + peers sidebar in `ServerLogs`.

### Verticals

Verticals are free-form strings (e.g. "dev", "design", "qa", "frontend"). There is no fixed hierarchy.

## Key Conventions

- **Runtime**: Bun only ‚Äî uses `bun:test`, native WebSocket
- **IDs**: UUID v4 via `crypto.randomUUID()`
- **Timestamps**: ISO strings via `new Date().toISOString()`
- **Protocol**: 4 message payload types in `shared/src/messages.ts` (query_ask, query_respond, ack, error), wrapped in `Envelope` (messageId, fromNode, toNode, projectId, timestamp, payload)
- **Config**: `ccom-config.json` for node identity + server URL. `node.projects` is an array (a node can belong to multiple projects). `.mcp.json` auto-generated by wizard.

## Adding a New MCP Tool

1. Add schema to `TOOL_DEFINITIONS` array in `channel.ts`
2. Add case to `handleCall()` switch in `channel.ts`
3. If it handles network messages, add handler in `ws-handlers.ts`
4. Test in `channel.test.ts`

## Deployment & Publishing

All deployments are triggered by **push to `main`** ‚Äî never publish or deploy manually.

### @tini-works/ccom (packages/node)
- **CI**: `.github/workflows/release-node.yml` ‚Äî tests ‚Üí builds ‚Üí auto-bumps patch version ‚Üí publishes to **npm** (public) ‚Üí creates git tag `ccom@X.Y.Z` + GitHub Release
- **Build**: `bun run build:node` bundles `@inv/shared` inline, keeps `@modelcontextprotocol/sdk` external ‚Üí `dist/cli.js` (with `#!/usr/bin/env bun` shebang)
- **CLI subcommands**: `init` (wizard), `serve` (MCP server), `update` (clear cache)

### Server + Admin (packages/server, packages/admin)
- **Deploy**: Dokploy at `inv-server.apps.quickable.co` ‚Äî multi-stage Docker build
  - Stage 1: builds admin React SPA with pnpm (Node 22)
  - Stage 2: runs server with Bun, serves admin at `/admin`
  - Admin UI at `/admin` ‚Äî public self-registration (no auth) or full management (with ADMIN_KEY)
- **Docker**: `packages/server/Dockerfile` + `docker-compose.yml` (Redis + server)
- **Config**: `.dokploy.json` in `packages/server/`
- **Port**: 4400, requires `ADMIN_KEY` env var and Redis
- **Admin UI sections**: Metrics ‚Üí Projects ‚Üí Nodes ‚Üí Signal Flow (React Flow graph) ‚Üí Server Logs (with live peers sidebar)

### Dashboard (packages/dashboard)
- **Framework**: Astro SSR with `@astrojs/node` adapter (standalone mode)
- **Build**: `astro build` ‚Üí deployable Node.js server

### Web (web/)
- **Framework**: Astro static site
- **Docker**: `web/Dockerfile` ‚Äî builds with Node, serves with Nginx on port 80
